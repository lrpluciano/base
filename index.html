Nível 1: Estrutura

1. O Mapa Mental de um Array

O que é na prática?

Imagine que você comprou um terreno e dividiu em 12 lotes iguais. Em Java, um Array é exatamente isso: um espaço contínuo na memória RAM que você reserva de uma vez só.

Por que não usar Listas dinâmicas? Listas crescem sozinhas, o que é cômodo, mas "caro" para o computador. O Array é fixo e extremamente rápido, ideal para sistemas de alta performance.

A Regra de Ouro do Índice

O computador sempre começa a contar do Zero. Por quê? Porque o índice representa o "deslocamento" do início do array.

  • Mês 1 (Janeiro): Deslocamento 0.
  • Mês 12 (Dezembro): Deslocamento 11.

Passo a Passo da Validação

  • Varredura: Use um laço for para olhar cada "lote" do seu array.
  • Filtro de Integridade: Antes de somar, pergunte: vendas[i] < 0?. Se sim, o dado é "lixo eletrônico" e deve ser rejeitado para não poluir sua média.
Nível 2: Inteligência

2. A Arte de Ordenar (Bubble Sort)

Por que ordenar na mão?

Usar Arrays.sort() é fácil, mas entender o Bubble Sort ensina você a manipular a memória. É a diferença entre saber dirigir e saber como o motor funciona.

A Mecânica do "Swap" (Troca)

Imagine que você tem um copo com suco (A) e um com água (B). Para trocar os líquidos, você precisa de um terceiro copo vazio (Temp).

// O processo de troca passo a passo
double temp = vendas[j]; // Passo 1: Copo vazio recebe Suco
vendas[j] = vendas[j+1]; // Passo 2: Copo de Suco agora recebe Água
vendas[j+1] = temp; // Passo 3: Copo de Água recebe o Suco do copo vazio

Otimização do Laço

A cada rodada do Bubble Sort, o maior número "sobe" para a última posição. Por isso, no próximo laço, você não precisa mais olhar para o último elemento. Isso economiza processamento!

Nível 3: Multidimensão

3. Matrizes: A Tabela de Dados

Uma matriz [5][4] nada mais é do que 5 arrays de 4 posições empilhados.

Col 0 (Tri 1) Col 1 (Tri 2) Col 2 (Tri 3) Col 3 (Tri 4)
Vendedor 0[0][0][0][1][0][2][0][3]
Vendedor 1[1][0][1][1][1][2][1][3]
Vendedor 2[2][0][2][1][2][2][2][3]

Soma por Linha (Vendedor)

Fixamos o i (Vendedor) e variamos o j (Trimestre). É como ler uma linha do livro por vez.

Soma por Coluna (Trimestre)

Fixamos o j (Trimestre) e variamos o i (Vendedor). É como ler uma coluna de jornal, de cima para baixo.

Nível 4: Desafio Mestre

4. Rankings e Sincronia de Dados

Este é o ponto onde muitos falham: ao ordenar os valores de venda, você precisa manter o nome do vendedor "colado" no valor.

Técnica de Vetores Paralelos

Imagine dois trilhos de trem. No trilho A estão os Valores. No trilho B estão os IDs dos Vendedores.

A Regra de Ouro: Se você mover um vagão no trilho A, o vagão correspondente no trilho B **deve** se mover exatamente para a mesma posição. Caso contrário, você dirá que o Vendedor 1 ganhou 50 mil, quando na verdade esse valor era do Vendedor 3.

  • Crie double[] totais e int[] indices.
  • No if do seu Bubble Sort, quando houver a troca de totais, repita a mesma lógica de troca para indices.
